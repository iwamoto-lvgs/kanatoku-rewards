# 引き継ぎ: かなトク進捗管理アプリ — 画面デザイン

## 概要
かなトク（神奈川県キャッシュレス還元キャンペーン）の還元上限を、**決済サービスごと**に管理するアプリ。複数の決済アプリ併用時に「どのサービスであと何円分還元を受けられるか」を横断把握し、取りこぼし・無駄打ちを防ぐ。サーバー不要・単一HTML・localStorage永続化が前提（詳細は同梱の `IMPLEMENTATION_SPEC.md` 参照）。

本READMEは、同梱の **デザインリファレンス（HTML）** を実装に落とすための仕様書です。**機能・データ・計算ロジックの確定仕様は `IMPLEMENTATION_SPEC.md` が正本**であり、本READMEは**ビジュアル/レイアウトの確定仕様**を補完します。両者を併せて参照してください。

## デザインファイルについて
同梱の `かなトク進捗管理アプリ.dc.html` は **HTMLで作成したデザインリファレンス**（意図する見た目と挙動を示すプロトタイプ）であり、そのまま本番コードとして使うものではありません。タスクは、このデザインを **対象コードベースの環境（バニラHTML/CSS/JS。`IMPLEMENTATION_SPEC.md` の非機能要件によりフレームワーク不使用）で再現すること**です。

> 注: `.dc.html` は社内デザインツールのプレビュー形式です。ブラウザで直接開けない場合は、ファイル内の `<x-dc>` 〜 `</x-dc>` 間のマークアップ（インラインスタイル付き）を見れば、各画面の正確なレイアウト・色・寸法が読み取れます。スクリーンショットが必要な場合は依頼してください（既定では同梱していません）。

## 忠実度
**ハイファイ（hifi）**。最終的な配色・タイポグラフィ・余白・状態を含むピクセル単位のモックです。下記の正確な値（hex・px・font-weight）でUIを再現してください。ただし対象は**モバイルファースト（実機は単一カラムのスマホ幅）**。デザインは4画面を横並びキャンバスに配置した「レビュー用」レイアウトであり、**実装は1画面ずつのスマホUI**として作ります（横並びキャンバスは再現不要）。

## 画面 / ビュー

実装は **下部タブバー**（ホーム / 記録 / 履歴 / 設定）で4画面を切り替える構成。各画面上部にはステータスバー風の余白（実機では不要、OSのものを使用）。

### 1. ダッシュボード（ホーム / メイン画面）
- **目的**: アクティブユーザーの全6サービスの残枠を一覧把握。
- **レイアウト**（上から縦積み）:
  1. **ヘッダー**: 左に「アクティブユーザー」ラベル（11px 弱色）+ ユーザー名（21px/800）+ 下シェブロン（タップで切替）。右に丸アバター（40px、アクセント背景、頭文字白）。
  2. **総付与額サマリーカード**: アクセント背景（`#1565C0`）、角丸22、白文字。ラベル「総付与額（このユーザー）」/ 金額 36px「¥7,300」+「/ ¥15,000」/ 白プログレスバー（= 総付与/15,000）/「全6サービス上限まで ◯%」。
  3. セクションラベル「サービス別の残枠」。
  4. **サービスカード ×6**（白、角丸18）。各カード:
     - 上段: ロゴタイル(40px 角丸11 サービス色 白文字) + サービス名(15/700) + 「付与済 ¥X / ¥2,500」(12 弱色)。右端に **残りバッジ**（"残り" + 金額15/800）。
     - **上限到達サービス**は右端を緑の **「✓ 上限達成」バッジ**（背景 `#E7F5EE`、文字 `#1E8E5A`）に置換し、残り表示は出さない。
     - プログレスバー（= 付与済/2,500、サービス色）。
     - **目安ボックス**（薄背景 `#F7F9FC`、角丸11）: 「残枠を使い切る目安支払額」+「20%店 ¥A」「10%店 ¥B」。**上限到達カードでは非表示**。
  5. **下部タブバー**（ホームがアクティブ=アクセント色、他は `#9AA4B2`）+ ホームインジケータ。

### 2. 支払いを記録
- **目的**: 1件の支払いを入力し、実付与額を自動計算。丸め・上限到達を明示。
- **レイアウト**:
  1. 画面タイトル「支払いを記録」(22/800)。
  2. **決済サービス選択**: 3列グリッドの6チップ。未選択は白+`#E6EAF0`枠+弱色文字、各チップ左に9pxのサービス色ドット。**選択中**はアクセント枠 + `rgba(21,101,192,.07)`背景 + 主色文字。
  3. **支払額**: 大きめ入力枠（アクセント枠）。「¥」(18 弱色) + 数値(28/800) + キャレット。
  4. **還元率トグル + 日付**を横2分割。還元率はセグメント（20% / 10%、選択側=白+影、`#EEF1F5`トラック）。日付は枠付きフィールド + カレンダーアイコン。
  5. **メモ**（任意）: 枠付きテキスト。
  6. **実付与額プレビューカード**（白、角丸16）:
     - ラベル「実付与額（自動計算）」+ 金額(32/800、アクセント色)。
     - 破線区切り下に内訳3行: 「理論付与（¥8,000 × 20%）→ ¥1,600」「1回上限を適用 → ¥1,500（橙文字）」「サービス残枠を適用（PayPay 残¥1,000）→ ¥1,000」。
     - **丸め通知（橙バナー）**: 三角警告アイコン +「1回上限により ¥1,500 に丸められました」。
     - **上限到達通知（赤バナー）**: 丸インフォアイコン +「この記録で PayPay は上限（¥2,500）に達します」。
     - ※両バナーは**該当する場合のみ**表示（丸め発生時/上限到達時）。
  7. **「記録する」ボタン**（アクセント背景、白文字、15/800、角丸14）。
  8. ホームインジケータ。

### 3. 履歴
- **目的**: 記録を日付降順で一覧。サービス・ユーザーでフィルタ。
- **レイアウト**:
  1. タイトル「履歴」。
  2. **フィルタチップ行**: 「すべてのサービス ▾」「ゆうき ▾」（ドロップダウン式ピル、99px角丸）。選択中のユーザーチップはアクセント枠+薄背景。
  3. **日付グループ**: 「6月26日 (金)」等の見出し(12 弱色) ごとに白カード（角丸16）。各行:
     - ロゴタイル(38px) + サービス名(14.5/700) + 「メモ・還元20%」(12 弱色、省略表示) + 右端に 支払額(14.5/700) と **付与額(+¥X、緑/800)**。
     - 行間は `#F1F4F8` のハイライン区切り。
  4. ホームインジケータ。

### 4. 設定（ユーザー管理・バックアップ）
- **目的**: ユーザーの追加/編集/削除・切替、JSONバックアップ、注意喚起、要項表示。
- **レイアウト**:
  1. タイトル「設定」。
  2. **ユーザー** グループカード: 各ユーザー行（丸アバター + 名前 + 使用中バッジ or 編集鉛筆アイコン）。最終行「＋ ユーザーを追加」（アクセント色）。
  3. **データ（バックアップ）** グループカード: 「JSONエクスポート」（ダウンロードアイコン+右シェブロン）/「JSONインポート」（アップロードアイコン+右シェブロン）。
  4. **注意バナー**（橙）: 「データは端末内（localStorage）にのみ保存されます。…定期的なエクスポートを推奨します。」
  5. **キャンペーン要項** カード: 還元率 20%/10%、サービス上限 ¥2,500、1回上限 ¥1,500、期間。
  6. ホームインジケータ。

## インタラクション & 挙動
- **ユーザー切替**: ヘッダーのユーザー名タップで選択。切替で全集計・一覧・入力既定が対象ユーザーに連動。
- **記録の自動計算**: 入力（サービス/額/率）変更のたびにプレビューを再計算（`IMPLEMENTATION_SPEC.md §2` の順序固定の純粋関数）。
- **丸め/上限の明示**: 1回上限で丸められた場合・サービス上限に達した場合は、プレビュー内バナーで必ずユーザーに通知。
- **編集/削除**: 履歴行から記録を編集・削除。変更時は累計・残枠・各カードを再計算。
- **フィルタ**: 履歴のサービス/ユーザーチップで絞り込み。
- **エクスポート/インポート**: JSONでバックアップ・復元。
- **レスポンシブ**: モバイルファースト。PCでも中央寄せ単一カラム or 適度な最大幅で破綻しないこと。
- 状態遷移・受け入れ基準の詳細は `IMPLEMENTATION_SPEC.md §3 / §9` を参照。

## 状態管理 / データ
- データモデル（`User` / `Payment`）・サービスID・保存方針（localStorage、`kanatoku:` 接頭辞、`loadData()`/`saveData()` ラッパー）は **`IMPLEMENTATION_SPEC.md §4・§5・§7` が正本**。
- 計算は保存層から独立した純粋関数として実装し、定数（2500 / 1500 / 0.20 / 0.10 / サービス一覧）はコード冒頭に集約（`§2`）。
- **派生表示値**: サービス残枠 = `2500 − 実付与累計`、目安支払額 = `残枠 ÷ 0.20`（20%店）/ `残枠 ÷ 0.10`（10%店）。総付与額 = 全サービス実付与累計の合計（最大 ¥15,000）。
- 受け入れ用テストケースは `§2` の表を必ず通すこと。
